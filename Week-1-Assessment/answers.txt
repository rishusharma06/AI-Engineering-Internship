Q1. (Answer)--
List 
- list is an ordered, mutable datatype in python which allows duplicates.
- useful when data may change frequently. ex- marks of students.

Tuple
- ordered, immutable datatype in Python, Duplicate elements are allowed.
- useful when data should not change. ex- RGB colors.

Set
- unordered collection of unique elements
- mainly used when we need duplicate removal ex- finding unique usernames from a list

Dict
- stores data as key–value pairs.
- used for mapping related information ex- storing student names with their marks.

def read_json_safely(path):
    try:
        #Opening file and loading data from json
        with open(path, "r") as file:
            return json.load(file)
    except FileNotFoundError:
        #if file not exists
        return {}
    except json.JSONDecodeError:
        #if JSON format is invalid
        return None

Q7. (Answer)--
Hard-coding an API key into a Python file is unsafe because if the code is uploaded to GitHub or shared 
then anyone can misuse the key. This can lead to security risks, unauthorized usage.
A better approach is to store sensitive values inside a .env file, which keeps secrets separate from the main code. 
Then we use the python-dotenv package to load those environment variables safely into the program.
The function used to load the .env file is: load_dotenv()
reading the API key: api_key = os.getenv("OPENAI_API_KEY")

from pydantic import BaseModel, Field
from typing import List

class WeatherReport(BaseModel):
    city: str = Field(...,min_length=1)
    temperature_c: float = Field(...,ge=-50, le=60)
    humidity: int = Field(...,ge=0, le=100)
    conditions: List[str]

valid example =
report = WeatherReport(
    city="Jaipur",
    temperature_c=31.4,
    humidity=42,
    conditions=["sunny", "windy"]
)

invalid example = 
bad_report = WeatherReport(
    city="",
    temperature_c=75,
    humidity=120,
    conditions=[1, 2]
)

Props are values passed from a parent component to a child component, and they are read-only.
They are used when one component needs to send data to another component.
Example: passing a user name to a ProfileCard component using <ProfileCard name="Rishu" />.

When the state changes, React automatically updates the UI. 
Example = a counter increasing when a button is clicked is managed using state.


Q14. (Answer)--
import React, { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  //Increase count
  const increment = () => {
    setCount(count + 1);
  };

  //Decrease count but never below 0
  const decrement = () => {
    if (count > 0) {
      setCount(count - 1);
    }
  };

  //Reset count to 0
  const reset = () => {
    setCount(0);
  };

  return (
    <div>
      <h2>Count: {count}</h2>

      <button onClick={increment}>+1</button>

      <button onClick={decrement}>-1</button>

      <button onClick={reset}>Reset</button>
    </div>
  );
}

export default Counter;

import React, { useState, useEffect } from "react";

function SearchableList() {
  const [users, setUsers] = useState([]);
  const [search, setSearch] = useState("");
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState("");

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then((response) => {
        if (!response.ok) {
          throw new Error("Failed to fetch users");
        }

        return response.json();
      })
      .then((data) => {
        setUsers(data);
        setLoading(false);
      })
      .catch((err) => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  const filteredUsers = users.filter((user) =>
    user.name.toLowerCase().includes(search.toLowerCase())
  );

  return (
    <div>
      <h2>User List</h2>

      <input
        type="text"
        placeholder="Search users..."
        value={search}
        onChange={(e) => setSearch(e.target.value)}
      />

      {loading && <p>Loading...</p>}

      {error && <p>Error: {error}</p>}

      {!loading && !error && filteredUsers.length === 0 && (
        <p>No users found</p>
      )}

      {!loading && !error && filteredUsers.length > 0 && (
        <ul>
          {filteredUsers.map((user) => (
            <li key={user.id}>
              {user.name}
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}

export default SearchableList;
